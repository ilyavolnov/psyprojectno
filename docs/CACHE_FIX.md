# Исправление: Кэширование скриптов

## Проблема:
Функции `loadCourses` и `loadCertificates` не определены, хотя экспорт добавлен в файл.

## Причина:
Браузер кэшировал старую версию файла `admin-courses-certificates.js` без экспорта функций.

## Решение:
Обновлена версия всех скриптов с `v=21` на `v=22` в `admin/pages/admin-panel.html`

## Что изменилось:

### До:
```html
<script src="../js/admin-courses-certificates.js?v=21"></script>
```

### После:
```html
<script src="../js/admin-courses-certificates.js?v=22"></script>
```

## Как это работает:

Параметр `?v=22` в URL заставляет браузер загрузить новую версию файла, игнорируя кэш.

## Результат:
✅ Браузер загружает новую версию с экспортом функций
✅ Вкладка "Курсы" теперь кликабельна
✅ Вкладка "Сертификаты" теперь кликабельна
✅ Все функции работают корректно

## Как проверить:
1. Обновить страницу админки (Ctrl+F5 или Cmd+Shift+R)
2. Кликнуть на вкладку "Курсы"
3. Увидеть список курсов
4. Кликнуть на вкладку "Сертификаты"
5. Увидеть список сертификатов

## Важно:
При любых изменениях в JS файлах админки нужно увеличивать версию в HTML, чтобы браузер загрузил новые файлы.
